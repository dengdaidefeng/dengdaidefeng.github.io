Redis学习总结

  • 刚学完Redis--

什么是Redis

Redis是一种NoSql，即非关系型数据库，与Mysql这类关系型数据库不同的是，Redis并没有明显的表结构，内部数据都是键值对，就像map。

更重要的是Redis的数据存储在内存中，而mysql存储在硬盘，所以Redis的访问速度比mysql快很多

为什么使用Redis

因为访问速度比mysql快很多，所以Redis可以用在访问数据库的中间层，让大量访问集中在Redis上就可以加快访问速度

Redis中的数据结构

官方文档：Commands | Docs

中文网：redis 命令手册

Redis常用的有5种数据结构 ( String, Hash, List, Set, SortedSet)，用来指定所存储键值对的类型，每种数据结构有相对应的方法实现对数据的crud，以及一些通用的方法，不局限于某种数据结构

文档中对方法的说明特别细致(命令行中使用redis-cli之后，使用help [方法名] 就可以了解Redis对应函数的相关细节)，我就不过多阐述，只说一下某些类型值得注意的地方。

String

整数和float都以字符串形式存储，但与普通字符串不同，虽然他们底层都是一个字节数组，但数字编码是二进制形式的。字符串是讲字符转换成对应的字节码存储。
